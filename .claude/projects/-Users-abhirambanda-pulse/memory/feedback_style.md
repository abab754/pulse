name: Feedback Style
description: User wants test+commit after each change, concrete advice, no overbuilding
type: feedback
---

Test and commit after every key change (minor or major). Be concrete and decisive — no generic advice. Keep the user from overbuilding.

**Why:** Fast MVP sprint with clean git history. User values professional engineering practices.

**How to apply:** After each implementation step, verify it works, then commit. Recommend cuts when scope creeps.
